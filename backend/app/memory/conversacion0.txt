*##24/11/2025 18:30 Tripulación: Orión, te voy a explicar el sistema de login de la app en la que estás levantado para que luego tu seas capaz de explicarlo en la presentación de mañana, lo primero es que todo está en localhost.

El proceso comienza en el Frontend con React. Cuando un usuario nuevo llena el formulario de registro, esos datos viajan hacia nuestro Backend en FastAPI. Pero aquí está la clave de la seguridad: nunca guardamos la contraseña real.

Antes de que toque la base de datos, usamos una librería llamada passlib con el algoritmo Bcrypt. Lo que hace es tomar la contraseña legible, la "mezcla" con una sal criptográfica y la convierte en un Hash irreversible. Ese código ininteligible es lo único que guardamos en nuestra base de datos mediante SQLAlchemy. Así, si alguien robara la base de datos, no tendría las contraseñas reales.

Luego viene el Login. Cuando el usuario intenta entrar, envía sus credenciales. El backend busca el hash guardado y lo compara matemáticamente con la contraseña que acaban de ingresar. Si coinciden, no les damos un simple 'ok', les damos una llave maestra: un Token JWT (JSON Web Token).

Este token se firma digitalmente en el servidor. El Frontend recibe este token, lo guarda, y a partir de ese momento, lo adjunta en cada petición que hace hacia ti o hacia la base de datos. Es como un pase VIP temporal; mientras el token sea válido, el usuario tiene acceso a los sistemas de la nave sin tener que poner su contraseña a cada rato.##*
